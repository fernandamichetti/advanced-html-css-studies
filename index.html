Section 4: Document Sections

How we divide document sections

💡 Sectioning root - A node in the tree that represents the root of a new section of the document

Self-Contained Composition

The Article Element

The Article Element represents a complete, or self-contained, composition in a document, page, application, or site and that’s, in principle,independently distributable or reusable (you can take that element to another place (another site, etc) and it will still make sense in itself). It’s self-contained

It falls into 3 main categories:

  1. Is a flow element - essentially the document flowing from top to bottom
  2. Is a sectioning element - Is a content that defines the scope of the outline, potentially has a heading and a outline.
  3. Is a palpable content - It’s expected, generally, that there’s something inside the element. There’ll be other elements inside of it.

💡 Inside the article element it can have any kind of flow content

The Section Element

Thematic Grouping

Represents a generic section of a document. In this context, is a thematic grouping of content, typically with a heading
Everything inside the element section must be intertwined, connected.

The Aside Element

Tangential Content

Represents a section of a page that consists of content that is tangentially related to the content around the aside element, and which could be considered separate from that content. Such sections are often represented as sidebars in printed typography.

💡 It's important to NOT think about the aside element as something visual. When you're authoring a HTML document you must do it in a future proof way, as in you don't know if one day this document will be read out load or printed, so the tags must be placed as a way to markup something, regardless of where it will appear in the page. 

                
                    <p>
                    My name is Fernanda and I'm learning how to author a HTML Document. <aside>I have two dogs, named Lupin and
                    Jager</aside>
                    </p>

The way that we know that it's allowed to have a article or section element inside a section element is through the documentation. When we look at the specification of those elements, we see that the content model says that it supports flow content and a aside element is a flow content. Therefore, this is legal/allowed!

Heading and Rank

Its content model is Phrasing, so it's different from the elements seen above. It means you can't put a flow element (as aside, article, section etc) inside it.
These elements have a rank given by the number in their name. The <h1> element has the highest rank, while the <h6> element has the lowest rank.

💡 When you have two heading of equal or higher rank, it's implied you're starting a new section and when you have a heading of lower rank it's implied that this new heading is a subsection of the previous one

Headers and Footers

Its content model is flow content, but with no header or footer

Header

The header element is intended to usually contain the section's heading, but it's not required. It can also be used to wrap a section's table of contents, a search form, or any relevant logs.

Footer

The footer element represents a footer for its nearest ancestor sectioning content or sectioning root element. Tipically contains information about its section such as who wrote it, links, copyright data and the like.

The Address element

Represents the contact information for its nearest article or body ancestor - not sibilings. It must not be used to represent arbitrary addresses, unless those addresses are in fact relevant contact information.

Its content model is also flow content, but with no heading content descendants, no sectioning content descendants and no <header>, <footer> or <address> element descendants

Section 5: Grouping Things

About HTML elements that have to do with grouping things

Paragraph

A paragraph, represented by the tag <p> is typically a run of phrasing content that forms a block of text with one or more sentences that discuss a particular topic, bun can also be used for more general thematic grouping.

Quotes

A quote, represented by the tag <blockquote> represents a section that is quoted from another source and the address to that source may be cited in the <cite> atribute
Example:

Attribution for the quotation, if any, must be placed outside the blockquote element.

- HTML Spec

💡The <blockquote> should be used when you want to cite a long phrase, instead the <q> tag should be used for inline quotations.

Unordened Lists

A unordened list, represented by the tag <ul>, represents a list of items, where the order is not important.
The itens inside the list will be represented by the tag <li>.

💡 An <li> element's end tag can be omitted if the <li> element is followed by another one or if there's no more content in the parent element

Ordened Lists

An ordened list, represented by the tag <ol> represents a list of items where the items have been intentionally ordered, which means that changing the order would change the meaning of the document.

💡 The <ol> tag supports a attribute called reversed. If present, it indicates that the list is descending (3, 2, 1..), if not, the list is ascending (1, 2, 3...)
Another attribute is the start that you can define in which number the list will start.

Association Lists

An association list, represented by the tag <dl> consists of zero or more name(<dt>)-value(<dd>) groups.
It can be terms and definitions, metadata topics and values, questions and answers or any other groups of name-value data.

dl

Open and closes a association list

dt

Representes the term or nome, part of a term-description group
When used inside a <dl> tag, the <dt> itself doesn't indicate that its content are a term being defined, for that you must use the <dfn> tag.

dd

Represents the description, definition or value, part of a term-description group in a description list (<dl>)

Example: 

                                
                                    <dl>
                                        <dt> Authors
                                        <dd> John
                                        <dd> Luke
                                        <dt> Editor
                                        <dd> Frank
                                    </dl>

💡 Even though the closing tag for <dd> and <dt> is optional, it's a good practice to use it, because its use prevents common brownser bugs with style sheet
