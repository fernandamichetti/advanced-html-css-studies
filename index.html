Section 4: Document Sections

How we divide document sections
💡 Sectioning root - A node in the tree that represents the root of a new section of the document

Self-Contained Composition

The Article Element

The Article Element represents a complete, or self-contained, composition in a document, page, application, or site and that’s, in principle,independently distributable or reusable (you can take that element to another place (another site, etc) and it will still make sense in itself). It’s self-contained

It falls into 3 main categories:

  • Is a flow element - essentially the document flowing from top to bottom
  • Is a sectioning element - Is a content that defines the scope of the outline, potentially has a heading and a outline.
  • Is a palpable content - It’s expected, generally, that there’s something inside the element. There’ll be other elements inside of it.
💡 Inside the article element it can have any kind of flow content

The Section Element

Thematic Grouping

Represents a generic section of a document. In this context, is a thematic grouping of content, typically with a heading
Everything inside the element section must be intertwined, connected.

The Aside Element - Tangential Content

Represents a section of a page that consists of content that's tangentially related to the content around the aside element, and which could be considered separate from that content. Such sections are often represented as sidebars in printed typography.
💡 It's important to NOT think about the aside element as something visual. When you're authoring a HTML document you must do it in a future proof way, as in you don't know if one day this document will be read out load or printed, so the tags must be placed as a way to markup something, regardless of where it will appear in the page. 

                    
                        <p>
                        My name is Fernanda and I'm learning how to author a HTML Document. <aside>I have two dogs, named Lupin and
                        Jager</aside>
                        </p>

💡 The way that we know that it's allowed to have a article or section element inside a section element is through the documentation. When we look at the specification of those elements, we see that the content model says that it supports flow content and a aside element is a flow content. Therefore, this is legal/allowed!

Heading and Rank

Its content model is Phrasing, so it's different from the elements seen above. It means you can't put a flow element (as aside, article, section etc) inside it.
These elements have a rank given by the number in their name. The <h1> element has the highest rank, while the <h6> element has the lowest rank.
💡 When you have two heading of equal or higher rank, it's implied you're starting a new section and when you have a heading of lower rank it's implied that this new heading is a subsection of the previous one

Headers and Footers

Its content model is flow content, but with no header or footer

Header

The header element is intended to usually contain the section's heading, but it's not required. It can also be used to wrap a section's table of contents, a search form, or any relevant logs.

Footer

The footer element represents a footer for its nearest ancestor sectioning content or sectioning root element. Tipically contains information about its section such as who wrote it, links, copyright data and the like.

The Address element

Represents the contact information for its nearest article or body ancestor - not sibilings. It must not be used to represent arbitrary addresses, unless those addresses are in fact relevant contact information.

Its content model is also flow content, but with no heading content descendants, no sectioning content descendants and no <header>, <footer> or <address> element descendants

A unordened list, represented by the tag <ul>, represents a list of items, where the order is not important.

An ordened list, represented by the tag <ol> represents a list of items where the items have been intentionally ordered, which means that changing the order would change the meaning of the document.

Section 6: Text itself

About changing the meaning of the document, but at an text level.
HTML tags/elements must not be used thinking visually, it's always about the meaning.

Emphasis - <em>

For emphasazing something in the text, the <em> tag is the most apropriate. The easiest way to know when it's appropriete to use <em> is to think how you would say the phrase.
💡 It must not be used as a way to make the text "italic". This tag, as well as the others seen bellow must be used thinking about the meaning of the document and not how it will be viewed. It's also not a tag that convey importance, for that the <strong> is the right element.
In the past it was normal to use the <i> tag when you wanted to make something italic, these days the <i> should be used when you want to represent a alternate voice, mood or otherwise offset from the normal prose, like indicating a different quality of text as a cientific name, technical term, an idiomatic phrase from another language etc.

Importance - <strong>

When you have something in the text that should be represented as something of strong importance, seriousness or urgency.
Even though by default most user agents make the <strong> tag appear with a bold effect, it's important to not use the element for this purpose.

Side Comments - <side>

It's a tag used to represent side comments such as small print, disclaimers, coyrights.
It should not be used for extented texts, but for short ones and also not be used for subheadings.

Line Breaks - br

The <br> tag must be used only in line breaks that are actually part of the content, not to separating thematic groups in a paragraph.

Section 7 - The Browner and the DOM

HTTP

  • Hypertext - Text that references other text which the user agent enables the user to immediately access
  • Protocol - A system of rules for two entities to communicate
  • Server - A computer or software that manages access to services and resources (documents, images etc)
  • Client - A computer or software that requests resources or services from a server
  • Hypertext Transfer Protocol - A set of rules for how two entities (client and server) can transfer text between themselves. Text which can link the user to other related text.
  • Rendering Engine - A computer program that transforms an HTML document into a visual, interactive representation of the document for the user

The HTML author will put the document in the server, then someone using a user agent (like a brownser) will request that HTML document (write the address of the site) and the server will deliver it. At that point the user agent will interpret that hypertext and deliver it to the user
That's why when you write a web address in the front there's "HTTP" or "HTTPS", cause that's the type of protocol you're using to see the site
Nowdays, besides Hypertext we send Hypermedia too though the HTTP

The <a> element defines a hyperlink, which is used to link from one page to another.
When used, the attribute href indicates the link's destination.
There's a few other attributes:

  • download - when the link is clicked the target will be downloaded
  • hreflang - specificies the language of the link
  • target - defines how the link will be open.
    • _blank
      • - opens in new link or tab
    • -parent
      • - opens in the same frame as it was clicked
    • _self
      • - opens in the parent frame

      • - opens in the full body of the window

Besides tags, there's attributes that can be used as an anchor in your HTML, like id.
ID is a global attribut that must be unique in the element's tree and must contain at last one character. It's a unique identifier to a fragment of the code. When you give a id to a tag, you can get to them by URL using the url with #(the standard for indentifying a fragment) and the id.

An object model that represents an HTML document, providing the ability to examine and change the document as presented via the user agent

Section 8 - Accessibility

Screen Reader

A screen reader is also a user agent, like a browser. But instead of traslating the data to a visual interface, the screen reader translates it to a non-visual way, through speech or braille.

Semantic and Accessibility

Using the right tags help the user agent screen reader to translate the data in the best way possible. For example, it can read a <h1> tag in a different volume or a slower voice, it can also be used to quickly navigate the document using keys for "jump to next heading" or "previous heading".

Aria - Accesible Rich Internet Applications

WAI-ARIA is a technical specification that provides a framework to improve the accessibility and interoperability of web content and applications.

Diference GET and POST

GET

Used to request data from a specified resource. The query string (name/value pairs) appears in the URL of the site.
It's only used to request data, not modify it and the data must be string type; It can be cached and remains in the browser history, besides that it can be bookmarked and have a lenght restriction (2048 characters).

💡 It never should be used when dealing with sensitive data, because the that is visible for everyone in the URL!

POST

Used to send data to a server to create/update a resource, supports different data types. The data sent via POST is stored in the request body of the HTTP request.
It can't be cached and don't reamin in the browser history, besides that it can't be bookmarked and don't have a lenght restriction.

Section 10: Stylesheets and Querying Trees

Inheritance

It's important to remember that when it comes to CSS, a rule that you apply to an element will not necessarily only apply to that said element, but sometimes it'll apply to its descendants too.

Universal Selector

Represented by *, it selects every element in the tree.

Attribute selectors

When you want to style a attribute. For that we put the name of the attribute inside [ ]. It can be used in different forms:

  • [name of attribute]: Every element that has an attribute will be given the style

    • 💡 It's important to remember that usually the <HTML> contains the attribute lang to define in which language the page is written and every descendant of element of

    <HTML>

    will inherite its styles
    Class, ID and slots are also attributes

  • [name of attribute=name of value]: Will only style the attribute whose value is axactly the name given. Example: [lang=pt-br]
  • [name of attribute~=name of value]: Will style attributes that have more than one word in which one is exactly the name given

Id selectors

When you want to style an specific ID you can use #nameOfID to set the rules of the style

Class selectors

To style an specific class you can use .nameOfClass. When you want to select of an specific element with that said class you can use element.nameOfClass. Example: div.names and even more than one class div.names.age that way if you have an element with the class name but not age the specified style won't be applied to that element

Group of selectors

When you want to give a rule of styles to more than one element you can use a comma to separate them. Example: ol, ul or even ol[lang], ul.name

💡 When using this to group elements it's important to garantee that everything is corect, because if there's something that doens't exist in the document even those who are correct won't have the style applied. For example, if you write incorrectly the class name

Descendants, Ancestors, Parents, Children and Sibilings

  • Descendant: Refers to any element that is connected but lower down the document tree - no matter how many levels
  • Ancestor: Refers to any element that is connected but further upthe document tree - no matter how many levels
  • Parent:Refers to an elements directly above and connected to an element.
  • Child:Refers to an elements directly below and conected to an element
  • Sibiling:Refers to an elements that shares the same parent to another element

Combinators: Descendant

Made up of two or more selectors separated by white space and selects elements B that are descendants from the element A.

Different from the next-sibiling combinator, this one selects sibiling elements no matter if it's directly or no, it just have to come after. To select this you have to use a ~

Pseudo-class

Selection based on information that lies outside the document tree - It means that instead of the author saying directly which element should be styled, the selector asks the browserto look at the document tree and the select document comes from that analysis

[...] classify elements on characteristics other than their name, attributes or content; in principle characteristics that cannot be deduced from the document tree. Pseudo-classes may be dynamic, in the sense that an element may acquire or lose a pseudo-class while a user interacts with the document.

                        li:nth-child(7)

It's possible to also choose a range of elements, from the x to the y the style must be applied:

                        li:nth-child(n+8):nth-child(-n+15)

:nth-last-child(An+B)

This pseudo-class matches elements based on their position among a group of sibilings, counting from the end

Type Child-Indexed Pseudo-classes

They work based on an element's indeix among elements of the same type in their sibiling list. So while the child-index looks for something around all the sibilings of a parent, no matter it's type, the type child-index looks the index of only the same tag of a sibling list.

:nth-of-type(An+B)

It does the same type as the nth-child, but only between those with the same tag inside a parent element.

Location Pseudo-Classes

This kind of pseudo classes lets you make selections based on links, it's status or where it may be going to.
The two most common (and mutually exclusive) are:

  • :link - Apply to links that have not yet been visited
  • :visited - Apply to those who have been visited by the user

User Action Pseudo-Classes

This pseudo-classes selects something based on a action that the user did.
The common are:

  • :hover or :focus - When the cursor hovers over a element or an elements hs the focus (using keyboard or mouse events)

Negation (Matches None) Pseudo-Class

Finds elements that are not something

                        section:not(#section1) {
                            font-style: italic;
                        }

It will style all the sections, except the section with the ID section1 with a italic style

Pseudo-elements

Basically elements in the document that haven't been marked up as elements. For example there's the pseudo-elements "first-letter", there isn't a tag that says it, but you can style it.
Example: 

                        p::first-letter {
                            font-weight: bold;
                        }

The Cascade

The Cascade means that after the declarations have been sorted (put in the right order), the one that will be applied is the top one and the bellow will be ignored.
This order takes into consideration:

  1. Origin
  2. Importance
  3. Specificity
  4. Order of Appareance

Origin

Besides the multiple stylesheet that the author can create, the user agent also have its own stylesheet.
Talking about the origin of the declarations, the author declaration will always be prioritize over the user agent declaration (if the user declaration is not marked up as important)

Importance

  1. Important user agent declarations
  2. Important author declarations
  3. Normal author declarations
  4. Normal user declarations

If the user agent marks something as important there's nothing the author can do.
The way that you can mark something as important is putting "!important" after the declarations.
Like this: 

                        h2 {
                            font-weight: bold !important;
                        }

The important mark up must only be used in specific cenarios.

Specificity

Each selector has a specificity number, that's defined by:

  1. Number of ID selectors in the selector
  2. Number of class, attributes and pseudo-classes in the selector
  3. Number of type selectors and pseudo-elements in the selector
  4. Ignore the universal selector
To discover the specifity number you have to count the 3 first options.
Example: 

                    #skills   100 (has one ID (1), there isn't any class, attribute or pseudo class (0) and there isn't any type selector or pseudo-element (0));

                    #skills .resume-list 110 (has one ID (1), one class (1) and there isn't any type selector (0));

                    .skills 010 (there isn't ID (0), has a class (1) e there isn't any type or pseudo-element (0));

                    section.skills 011 (there's no ID (0), has one class (1) and one type selector (1))

The higher number the higher the specificity;
