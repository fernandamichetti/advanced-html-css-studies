Section 4: Document Sections

How we divide document sections
💡 Sectioning root - A node in the tree that represents the root of a new section of the document

Self-Contained Composition

The Article Element

The Article Element represents a complete, or self-contained, composition in a document, page, application, or site and that’s, in principle,independently distributable or reusable (you can take that element to another place (another site, etc) and it will still make sense in itself). It’s self-contained

It falls into 3 main categories:

  1. Is a flow element - essentially the document flowing from top to bottom
  2. Is a sectioning element - Is a content that defines the scope of the outline, potentially has a heading and a outline.
  3. Is a palpable content - It’s expected, generally, that there’s something inside the element. There’ll be other elements inside of it.
💡 Inside the article element it can have any kind of flow content

The Section Element

Thematic Grouping

Represents a generic section of a document. In this context, is a thematic grouping of content, typically with a heading
Everything inside the element section must be intertwined, connected.

The Aside Element - Tangential Content

Represents a section of a page that consists of content that's tangentially related to the content around the aside element, and which could be considered separate from that content. Such sections are often represented as sidebars in printed typography.
💡 It's important to NOT think about the aside element as something visual. When you're authoring a HTML document you must do it in a future proof way, as in you don't know if one day this document will be read out load or printed, so the tags must be placed as a way to markup something, regardless of where it will appear in the page. 

                    
                        <p>
                        My name is Fernanda and I'm learning how to author a HTML Document. <aside>I have two dogs, named Lupin and
                        Jager</aside>
                        </p>

💡 The way that we know that it's allowed to have a article or section element inside a section element is through the documentation. When we look at the specification of those elements, we see that the content model says that it supports flow content and a aside element is a flow content. Therefore, this is legal/allowed!

Heading and Rank

Its content model is Phrasing, so it's different from the elements seen above. It means you can't put a flow element (as aside, article, section etc) inside it.
These elements have a rank given by the number in their name. The <h1> element has the highest rank, while the <h6> element has the lowest rank.
💡 When you have two heading of equal or higher rank, it's implied you're starting a new section and when you have a heading of lower rank it's implied that this new heading is a subsection of the previous one

Headers and Footers

Its content model is flow content, but with no header or footer

Header

The header element is intended to usually contain the section's heading, but it's not required. It can also be used to wrap a section's table of contents, a search form, or any relevant logs.

Footer

The footer element represents a footer for its nearest ancestor sectioning content or sectioning root element. Tipically contains information about its section such as who wrote it, links, copyright data and the like.

The Address element

Represents the contact information for its nearest article or body ancestor - not sibilings. It must not be used to represent arbitrary addresses, unless those addresses are in fact relevant contact information.

Its content model is also flow content, but with no heading content descendants, no sectioning content descendants and no <header>, <footer> or <address> element descendants

Section 5: Grouping Things

About HTML elements that have to do with grouping things

Paragraph

A paragraph, represented by the tag <p> is typically a run of phrasing content that forms a block of text with one or more sentences that discuss a particular topic, bun can also be used for more general thematic grouping.

Quotes

A quote, represented by the tag <blockquote> represents a section that is quoted from another source and the address to that source may be cited in the <cite> atribute

Attribution for the quotation, if any, must be placed outside the blockquote element.

💡The <blockquote> should be used when you want to cite a long phrase, instead the <q> tag should be used for inline quotations.

Unordened Lists

A unordened list, represented by the tag <ul>, represents a list of items, where the order is not important.
The itens inside the list will be represented by the tag <li>.
💡 An <li> element's end tag can be omitted if the <li> element is followed by another one or if there's no more content in the parent element

Ordened Lists

An ordened list, represented by the tag <ol> represents a list of items where the items have been intentionally ordered, which means that changing the order would change the meaning of the document.
💡 The <ol> tag supports a attribute called reversed. If present, it indicates that the list is descending (3, 2, 1..), if not, the list is ascending (1, 2, 3...)
Another attribute is the start that you can define in which number the list will start.

Association Lists

An association list, represented by the tag <dl> consists of zero or more name(<dt>)-value(<dd>) groups.
It can be terms and definitions, metadata topics and values, questions and answers or any other groups of name-value data.

dl

Open and closes a association list

dt

Representes the term or nome, part of a term-description group
When used inside a <dl> tag, the <dt> itself doesn't indicate that its content are a term being defined, for that you must use the <dfn> tag.

dd

Represents the description, definition or value, part of a term-description group in a description list (<dl>)

Example: 

                                    
                                        <dl>
                                            <dt> Authors
                                            <dd> John
                                            <dd> Luke
                                            <dt> Editor
                                            <dd> Frank
                                        </dl>

💡 Even though the closing tag for <dd> and <dt> are optional if the imeddiately following tag is another dd or dt, it's a good practice to use it, because its use prevents common brownser bugs with style sheet
💡 The tag <dl> are not allowed inside a <p> tag, cause it's not a phrasing content

Section 6: Text itself

About changing the meaning of the document, but at an text level.
HTML tags/elements must not be used thinking visually, it's always about the meaning.

Emphasis - <em>

For emphasazing something in the text, the <em> tag is the most apropriate. The easiest way to know when it's appropriete to use <em> is to think how you would say the phrase.
💡 It must not be used as a way to make the text "italic". This tag, as well as the others seen bellow must be used thinking about the meaning of the document and not how it will be viewed. It's also not a tag that convey importance, for that the <strong> is the right element.
In the past it was normal to use the <i> tag when you wanted to make something italic, these days the <i> should be used when you want to represent a alternate voice, mood or otherwise offset from the normal prose, like indicating a different quality of text as a cientific name, technical term, an idiomatic phrase from another language etc.

Importance - <strong>

When you have something in the text that should be represented as something of strong importance, seriousness or urgency.
Even though by default most user agents make the <strong> tag appear with a bold effect, it's important to not use the element for this purpose.

Side Comments - <side>

It's a tag used to represent side comments such as small print, disclaimers, coyrights.
It should not be used for extented texts, but for short ones and also not be used for subheadings.

Line Breaks - br

The <br> tag must be used only in line breaks that are actually part of the content, not to separating thematic groups in a paragraph.

Example:

1. Wrong use of the <br> tag:

                
<p>
<label>Name: Fernanda</label>
<br>
<label>Age: 26</label>
</p>

Name: Fernanda
Age: 26

2. Right use to use <br> tag:

                
<p>
Name: Fernanda
</p>
<p>
Age: 26
</p>

Name: Fernanda

Age: 26

Is a meaningless element, but can be useful when used together with the global attributel like class, lang or dir

Section 7 - The Browner and the DOM

HTTP

  • Hypertext - Text that references other text which the user agent enables the user to immediately access
  • Protocol - A system of rules for two entities to communicate
  • Server - A computer or software that manages access to services and resources (documents, images etc)
  • Client - A computer or software that requests resources or services from a server
  • Hypertext Transfer Protocol - A set of rules for how two entities (client and server) can transfer text between themselves. Text which can link the user to other related text.
  • Rendering Engine - A computer program that transforms an HTML document into a visual, interactive representation of the document for the user

The HTML author will put the document in the server, then someone using a user agent (like a brownser) will request that HTML document (write the address of the site) and the server will deliver it. At that point the user agent will interpret that hypertext and deliver it to the user
That's why when you write a web address in the front there's "HTTP" or "HTTPS", cause that's the type of protocol you're using to see the site
Nowdays, besides Hypertext we send Hypermedia too though the HTTP

Anchor Tags and Hypertext - a

The <a> element defines a hyperlink, which is used to link from one page to another.
When used, the attribute href indicates the link's destination.
There's a few other attributes:

  • download - when the link is clicked the target will be downloaded
  • hreflang - specificies the language of the link
  • target - defines how the link will be open.
    • _blank
      • - opens in new link or tab
    • -parent
      • - opens in the same frame as it was clicked
    • _self
      • - opens in the parent frame
    • _top
      • - opens in the full body of the window

Besides tags, there's attributes that can be used as an anchor in your HTML, like id.
ID is a global attribut that must be unique in the element's tree and must contain at last one character. It's a unique identifier to a fragment of the code. When you give a id to a tag, you can get to them by URL using the url with #(the standard for indentifying a fragment) and the id.
Example:
www.fernandamichetti.com/#aboutme

Object

A collection of data (information) and code (that accomplishes things) which together represents something

Model

A representation of a thing

Object Model

A collection of objects that represent a thing and provide access to examine and change that thing

The Document Object Model - DOM

An object model that represents an HTML document, providing the ability to examine and change the document as presented via the user agent
💡 When you open the DevTools, the "Elements" tab is not showing the HTML, but something that looks like the HTML to make it easier for the developers to visualize, but the DOM. The HTML itself will be shown in the network > response tab.
The HTML document is downloaded by the User Agent who then creates the DOM based on the HTML that then is used to generate the visual representation of the HTML, which is the web page delivered to the user.
